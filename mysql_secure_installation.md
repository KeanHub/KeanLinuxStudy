# MySql的安装及安全初始化

1. 安装mysql和mysql-server。由于前面已经配置好yum源。直接使用
   令：yum install mysql -y，yum install mysql-server -y。其中mysql-server是服务器端，简单来说
   就是创建数据库的，mysql是mysql的客户端工具，简单来说就是操作数据库的。真实生产环境下，服务器端和客户端有可能是分开的，按需下载即可。
2. 启动mysql-server。使用命令：systemctl start mysqld，服务
   使用命令：mysql -uroot，前面提示符变为mysql>，说明登陆成功。另外还有一种登陆方式，使用命令：
   mysql -uroot -p，这里会提示输密码，默认是空密码或者生成随机密码。我就在这里介绍一下两种登陆方式，第一种后面不加-p，不输密码的方式，这是套接字连接，所谓的套接字连接，就是使用mysql的套接字文件/var/lib/mysql/mysql.sock进行验证身份，是本地免密连接方式；第二种输入密码的登陆方式就很好理解了就是使用mysql的密码登陆。那两种方式都有一定的安全隐患，第一种套接字登陆如果root用户密码被泄露，，直接绕过数据库密码接管数据库，第二种如果默认是空密码，那任意用户都可以登陆，所以我们这里对这两种登陆方式做一个安全初始化。
3. 安全初始化。使用命令：mysql_secure_installation，
   3.1 输入密码，有三种等级，0 = LOW：仅要求密码≥8 位，适合测试环境；1 = MEDIUM（生产常用）：长度≥8，需同时含大小写字母、数字、特殊符号；2 = STRONG：在中等强度基础上，额外拦截常见弱密码。如果不满足密码等级要求，会要求重新输入。
   3.2 是否删除匿名用户，如果不关闭攻击者可以通过匿名用户非法接入数据库，建议关闭。
   3.3 是否关闭root远程登陆，如果开启远程登陆，那么攻击者可以通过外网通过root账号socket非法访问数据库，但是本地socket登陆无碍，建议关闭。下一项是否删除 test 数据库以及对应的访问权限。
   3.4 是否删除数据库的test数据库，test库默认开放访问权限，入侵者可借助这个库创建表、写入脏数据、占用数据库资源，作为攻击跳板，建议删除。
   3.5 是否现在重载权限表，简单来说就是否让前面所有安全配置改动立刻生效，建议重置。
4. 到这里安全初始化就成功了，尝试登陆数据库，两种方法均可。
   4.1 但是你也发现了，本地的root用户通过socket可以免密登陆，似乎很危险，虽然限制了远程root登陆，但如果攻击者并不直接远程root登陆，而是通过先登陆跳板机，跳板机是物理上的本地终端，那么就可以直接以通过socket直接登陆数据库，这是非常危险的。
   4.2 我建议修改root用户账号的认证方式为mysql_native_password，首先登陆mysql，使用命令：ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY '密码';退出数据库，再执行 mysql -uroot，会直接报错拒绝登录；必须用 mysql -uroot -p 输入密码才可登录，socket 免密就已经失效。
